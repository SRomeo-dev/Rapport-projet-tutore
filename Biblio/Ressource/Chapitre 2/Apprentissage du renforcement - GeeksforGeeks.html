    Apprentissage du renforcement

    Dernière mise à jour : 18 avril, 2023
    Améliorer

    L'apprentissage par renforcement est un domaine de l'apprentissage automatique. Il s'agit de prendre des mesures appropriées pour maximiser la rémunération dans une situation particulière. Il est utilisé par divers logiciels et machines pour trouver le meilleur comportement possible ou le meilleur chemin qu'il devrait prendre dans une situation spécifique. L'apprentissage par le renforcement diffère de l'apprentissage supervisé de telle sorte que, lors de l'apprentissage supervisé, les données de formation ont la clé de réponse avec elle, de sorte que le modèle est formé avec la bonne réponse elle-même, alors que dans l'apprentissage du renforcement, il n'y a pas de réponse, mais l'agent de renforcement décide de quoi faire pour accomplir la tâche donnée. En l'absence d'un ensemble de données de formation, il est tenu de tirer les leçons de son expérience.

    L'apprentissage par renforcement (RL) est la science de la prise de décision. Il s'agit d'apprendre le comportement optimal dans un environnement pour obtenir une récompense maximale. En RL, les données sont accumulées à partir de systèmes d'apprentissage automatique qui utilisent une méthode d'essai et d'erreur. Les données ne font pas partie de l'entrée que nous trouverions dans l'apprentissage automatique supervisé ou non supervisé.

    L'apprentissage par renforcement utilise des algorithmes qui apprennent des résultats et décident de la prochaine action. Après chaque action, l'algorithme reçoit un retour d'information qui l'aide à déterminer si le choix qu'il a fait était correct, neutre ou incorrect. C'est une bonne technique à utiliser pour les systèmes automatisés qui doivent prendre beaucoup de petites décisions sans orientation humaine.

    L'apprentissage du renforcement est un système autonome et auto-apprenant qui apprend essentiellement par tâtonnements. Il effectue des actions dans le but de maximiser les récompenses, c'est-à-dire qu'il apprend en faisant les meilleurs résultats.

    Exemple:

    Le problème est le suivant : nous avons un agent et une récompense, avec de nombreux obstacles intermédiaires. L'agent est censé trouver le meilleur chemin possible pour atteindre la récompense. Le problème suivant explique plus facilement le problème.

    L'image ci-dessus montre le robot, le diamant et le feu. Le but du robot est d'obtenir la récompense qui est le diamant et d'éviter les obstacles qui sont tirés. Le robot apprend en essayant tous les chemins possibles, puis en choisissant le chemin qui lui donne la récompense avec les moindres obstacles. Chaque étape juste donnera au robot une récompense et chaque mauvais pas va soustraire la récompense du robot. La récompense totale sera calculée lorsqu'elle aura atteint la récompense finale qu'est le diamant.
    Principaux points de l'apprentissage du renforcement

    • Entrée: L'entrée devrait être un état initial à partir duquel le modèle commencera.
    • Produit: Il existe de nombreux résultats possibles car il existe toute une gamme de solutions à un problème particulier.
    • Formation: La formation est basée sur l'apport, le modèle retournera un État et l'utilisateur décidera de récompenser ou de punir le modèle sur la base de sa production.
    • Le modèle continue d'apprendre.
    • La meilleure solution est décidée sur la base de la récompense maximale.
     

    Différence entre l'apprentissage par renforcement et l'apprentissage supervisé:

    Types de renfort:

    Il existe deux types de renforcement:

    1. Positif:Le renforcement positif est défini comme lorsqu'un événement, se produit en raison d'un comportement particulier, augmente l'intensité et la fréquence du comportement. En d'autres termes, il a un effet positif sur le comportement.

      Les avantages de l'apprentissage par renforcement sont les suivants:

      • Maximiser les performances
      • Sustain Changement pendant une longue période
      • Trop de renfort peut conduire à une surcharge d'états qui peuvent diminuer les résultats.
    2. Négatif :Le renforcement négatif est défini comme le renforcement du comportement parce qu'une condition négative est arrêtée ou évitée.
      Avantages de l'apprentissage par renforcement:
      • Augmentation du comportement
      • Défier une norme minimale de performance
      • Il ne fournit que suffisamment pour atteindre le comportement minimum

    Éléments d'apprentissage renforcé

    Les éléments d'apprentissage du renforcement sont les suivants:

    1. Politique
    2. Fonction de récompense
    3. Fonction de valeur
    4. Modèle de l'environnement

    Politique: La politique définit le comportement de l'agent d'apprentissage pour une période donnée. Il s'agit d'une cartographie des états perçus de l'environnement aux mesures à prendre dans ces États.

    Fonction de récompense: La fonction de récompense est utilisée pour définir un objectif dans un problème d'apprentissage du renforcement. Une fonction de récompense est une fonction qui fournit un score numérique basé sur l'état de l'environnement

    Fonction de valeur: Les fonctions de valeur spécifient ce qui est bon à long terme. La valeur d'un État est le montant total de la récompense qu'un agent peut s'attendre à accumuler pour l'avenir, à partir de cet état.

    Modèle de l'environnement: Les modèles sont utilisés pour la planification.

    Problème d'affectation du crédit: Les algorithmes d'apprentissage de renforcement apprennent à générer une valeur interne pour les états intermédiaires quant à la qualité de leur réalisation à l'objectif. Le décideur d'apprentissage est appelé l'agent. L'agent interagit avec l'environnement qui inclut tout ce qui se trouve en dehors de l'agent.

    L'agent dispose de capteurs pour décider de son état dans l'environnement et prend des mesures qui modifient son état.

    Le modèle de problème d'apprentissage du renforcement est un agent interagissant en permanence avec un environnement. L'agent et l'environnement interagissent dans une séquence de pas de temps. A chaque pas de temps t, l'agent reçoit l'état de l'environnement et une récompense numérique scalaire pour l'action précédente, puis l'agent sélectionne ensuite une action.

    L'apprentissage par renforcement est une technique pour résoudre les problèmes de décision de Markov.

    L'apprentissage par renforcement utilise un cadre formel définissant l'interaction entre un agent d'apprentissage et son environnement en termes d'états, d'actions et de récompenses. Ce cadre est destiné à être un moyen simple de représenter les caractéristiques essentielles du problème de l'intelligence artificielle.

    Diverses applications pratiques de l'apprentissage de renforcement

    • RL peut être utilisé en robotique pour l'automatisation industrielle.
    • RL peut être utilisé dans l'apprentissage automatique et le traitement des données
    • RL peut être utilisé pour créer des systèmes de formation qui fournissent des instructions et du matériel personnalisés en fonction des besoins des étudiants.

    Application des enseignements de renforcement

    1. Robotique: Les robots à comportement préprogrammé sont utiles dans des environnements structurés, comme la chaîne de montage d'une usine de fabrication d'automobiles, où la tâche est de nature répétitive.

    2. Un maître joueur d'échecs fait un pas. Le choix est informé à la fois par la planification, l'anticipation des réponses possibles et des contre-réponses.

    3. Un contrôleur adaptatif ajuste les paramètres du fonctionnement d'une raffinerie de pétrole en temps réel.

    Le RL peut être utilisé dans de grands environnements dans les situations suivantes:

    1. Un modèle de l'environnement est connu, mais une solution analytique n'est pas disponible;
    2. Seul un modèle de simulation de l'environnement est donné (objet de l'optimisation basée sur la simulation)
    3. La seule façon de collecter des informations sur l'environnement est d'interagir avec lui.

    Avantages et inconvénients de l'apprentissage par le renforcement

    Avantages de l'apprentissage par le renforcement

    1. L'apprentissage par renforcement peut être utilisé pour résoudre des problèmes très complexes qui ne peuvent pas être résolus par des techniques conventionnelles.

    2. Le modèle peut corriger les erreurs qui se sont produites au cours du processus de formation.

    3. Dans RL, les données de formation sont obtenues via l'interaction directe de l'agent avec l'environnement.

    4. L'apprentissage du renforcement peut gérer des environnements non déterministes, ce qui signifie que les résultats des actions ne sont pas toujours prévisibles. Cela est utile dans les applications réelles où l'environnement peut évoluer au fil du temps ou est incertain.

    5. L'apprentissage de renforcement peut être utilisé pour résoudre un large éventail de problèmes, y compris ceux qui impliquent la prise de décision, le contrôle et l'optimisation.

    6. L'apprentissage par renforcement est une approche flexible qui peut être combinée avec d'autres techniques d'apprentissage automatique, telles que l'apprentissage profond, pour améliorer les performances.

    Inconvénients de l'apprentissage par renforcement

    1. L'apprentissage par renforcement n'est pas préférable à l'utilisation pour résoudre des problèmes simples.

    2. L'apprentissage par renforcement nécessite beaucoup de données et beaucoup de calculs

    3. L'apprentissage par le renforcement dépend fortement de la qualité de la fonction de récompense. Si la fonction de récompense est mal conçue, l'agent peut ne pas apprendre le comportement souhaité.

    4. L'apprentissage du renforcement peut être difficile à déboguer et à interpréter. On ne sait pas toujours pourquoi l'agent se comporte d'une certaine manière, ce qui peut rendre difficile le diagnostic et la résolution des problèmes.

    # Define the Q-table and learning rate
    q_table = np.zeros((state_size, action_size))
    alpha = 0.8
    gamma = 0.95
     
    # Train the Q-Learning algorithm
    for episode in range(num_episodes):
        state = env.reset()
        done = False
        while not done:
            # Choose an action
            action = np.argmax(
                q_table[state, :] + np.random.randn(1, action_size) * (1. / (episode + 1)))
     
            # Take the action and observe the new state and reward
            next_state, reward, done, _ = env.step(action)
     
            # Update the Q-table
            q_table[state, action] = (1 - alpha) * q_table[state, action] + \
                alpha * (reward + gamma * np.max(q_table[next_state, :]))
     
            state = next_state
     
    # Test the trained Q-Learning algorithm
    state = env.reset()
    done = False
    while not done:
        # Choose an action
        action = np.argmax(q_table[state, :])
     
        # Take the action
        state, reward, done, _ = env.step(action)
        env.render()
